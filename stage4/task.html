[TITLE]Practice 3155618[/TITLE]

Description

For now, we are mining blocks to create a blockchain, but just the blockchain itself is not particularly useful. The most useful information in the blockchain is the data that every block stores. The information can be anything. Let's create a simple chat based on the blockchain. If this blockchain works on the internet, it would be a worldwide chat. Everyone can add a line to this blockchain, but no one can edit it afterward. Every message would be visible to anyone.

Objectives

In this stage, you need to upgrade the blockchain. Each block, except the Genesis Block, should contain a single chat message received by the blockchain before starting the new block mining process. When the new block gets created, the message should become a part of the new block, and all the miners should start to search for a magic number for this block. Remember about using thread synchronization, as there is a lot of shared data.

You don't need any network connections, as this is only a simulation of the blockchain. Use a single blockchain and suppose that any user can send a single message to the blockchain just by invoking one method of the blockchain; this message should be further included within the newly mined blocks and printed as a new line "Block data:".

[ALERT-primary]

Note that the single chat message should include whitespaces. To read a string with whitespaces from the input, you'll need to implement a scanner using the functions from the bufio package.

[/ALERT]

So, the algorithm for adding a single chat message to the blockchain is the following:

  1. The Genesis Block's "Block data:" line should contain the string "No messages". The Genesis Block should NOT contain any data or message. After the Genesis Block is created and printed, your program should prompt the user: "Enter a single chat message to send to the Blockchain:".

  2. While miners search for a magic number to create the new block, the blockchain should temporarily keep the previously sent chat message in memory.

  3. After creating the new block, the previously sent chat message should be included in the new block, printed after the "Block data:" line, and then deleted from memory.

  4. The process will repeat from step 1 until the fifth and final block is output.

Example

To be tested successfully, the program must output the information of five (5) blocks in sequence.

The lines that start with > represent user input.
