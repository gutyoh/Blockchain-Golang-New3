[TITLE]Practice 3155615[/TITLE]

Description

After the blockchain gets initiated and the Genesis Block is created, the blockchain shouldn't create new/additional blocks. The blockchain just keeps the chain valid and accepts new blocks from outside.

In the outside world, there are a lot of computers that try to create a new block. All they do is search for a magic number to create a block whose hash starts with some zeros. The first computer to do so is a winner, the blockchain accepts this new block, and then all these computers try to find a magic number for the next block.

There is a special word for this: mining. The process of mining blocks is hard work for computers, like the process of mining minerals in real life is hard work. Computers that perform this task are called miners.

Note that the new blocks will get mined faster if there are more miners. But the problem is that we want to create new blocks with a stable frequency. For this reason, the blockchain should regulate the number N: the number of zeros at the start of a hash of the new block. If suddenly there are so many miners that the new block gets created in less than 5 seconds, the next block's complexity should increase by increasing the number N. On the other hand, if there are so few miners that creating a new block takes longer than 10 seconds, the number N should be lowered.

Objectives

In this stage, you should create a lot of goroutines with miners, and every one of them should contain the same blockchain. Remember that the Genesis Block gets created when the blockchain is initialized, so it isn't mined.

The miners should start mining new blocks after the Genesis Block is created, and the blockchain should regulate the number N. The blockchain should check the validity of the incoming block (ensure that the previous hash equals the hash of the last block of the blockchain and the hash of this new block starts with N zeros).

At the start, the number N equals 0. However, it should change according to the following conditions:

  • N should be increased by 1 if the block creation time is less than 5 seconds

  • N should be decreased by 1 if the block creation time is greater than 10 seconds

  • N should stay the same if the block creation time is greater or equal to 5 seconds but less or equal to 10 seconds.

[ALERT-warning]

Note that your program should not finish until it creates and outputs five (5) blocks in this stage. The tests check the output right after your program finishes execution.

[/ALERT]
